③ 关键分析与决策

A100 40G → 80G：还要不要量化？

结论：基本可彻底不量化。 A100（Ampere sm8.0）没有 FP8 张量核，量化只为"塞得下"、不加速。80G 显存足以原生 bf16/fp16 跑 Wan2.2-I2V-A14B（双专家各 ~28G），故直接 fp16 原生即上限最优；fp8 仅在显存不足时才有意义。据此把出片默认从 fp8 切到 fp16（提交 9291fef）。

为什么"不能 FP8"（硬件根因）

FP8 的硬件加速从 Hopper（H100，sm9.0）起才有，Ada（sm8.9，4090/L4）亦有；A100（sm8.0）没有。所以 A100 上加载 fp8 权重能省显存，但运算只能"反量化成 fp16 再算"（日志里的 emulated ops / manual cast）→ 不快、反因开销更慢、还撞 OOM。这是硬件代差，非配置开关。

即梦（Dreamina / Seedance）为何那么快

字节自研 Seedance，生产级集群：H100+ 原生 FP8 + 蒸馏少步专用模型 + 模型常驻零加载 + 千卡并行 + 定制推理栈（TensorRT/FA3/缓存）+ 低清→超分。公开口径：Seedance 2.0 Fast 出 5–10s 视频仅 30–60s，约 1 元/秒。与"单卡自托管研究档"是两个量级；自托管的价值在无审查 + 自有角色 LoRA + 免费不限量 + 完全可控。
