1 · 七步操作

1

建项目

在哪 面板入口 / 对 Agent 说工具 create_video_project

「建一个短剧项目，叫《xxx》」→ 进制作面板，后续全在面板点按钮。

2

配角色 (可选，建议)

工具 configure_character存 .agent/config.json

有人物 LoRA 就配触发词+LoRA 路径，出图自动注入，主角脸稳。没 LoRA 就留空(trigger_word:""、flux_lora:"none")，靠描述压一致性。

别套别人的 LoRA：config 里残留别的角色触发词/LoRA，会把每张图强行带成那个角色，跟你提示词无关。
3

加分镜 · 6 个字段

字段含义喂给谁
image_prompt画面长什么样FLUX 出图
motion_prompt镜头怎么动Wan 出片
narration旁白/台词普通镜头=画外音；对口型=人物台词
subtitle屏幕字幕烧屏(留空=沿用旁白)
对口型人物开不开口开=S2V，关=普通运镜
4

出图(FLUX)

点 每镜「出图」→参数卡防抽卡 一次 4 张挑一张竖屏 768×1024
FLUX-dev 读不懂中文。你写中文 image_prompt，系统出图前自动翻成英文再下发(对你隐形，IMAGE_PROMPT_AUTOTRANSLATE 默认开)。
出图吃满显存(FLUX 压线 24G) → 先停掉 ComfyUI 独占整卡。

▲ 出图参数卡：张数填 4 防抽卡，提示词写中文，点「出图」

5

选图

每镜出 N 张 → 点最满意那张的 ✓。不满意：重出图换种子；或上传图片用自己的图。

6

出片 · 按镜头类型分两路

普通镜头 Wan 图生视频

  • 选好图 → 行上段数填 N → 点「出视频」
  • 段数=尾帧接续：单段 ≈ 2 秒，N 段拼 ≈2N 秒
  • 不够长 → 「➕ 再续一段」反复加
  • 不会写运镜 → 「✨AI 推荐」据末帧给一句

对口型镜头 Wan2.2-S2V

  • 确认 narration 写的是台词
  • 勾「🗣 对口型」→「对口型出片」
  • 台词 → TTS → S2V → 口型同步(自带人声)
  • 时长跟台词走、不截断(帧数按音频算)

▲ 普通镜头：选图 ✓ → 填段数 → 出视频

▲ 对口型镜头：勾 🗣 对口型 → 按钮变「对口型出片」

S2V 一口气整段生成(非接续)，长度由音频定；帧数=⌈台词秒数×fps⌉ 取 Wan 合法 4n+1，口型对到最后一字。
对口型必须先启动 ComfyUI，否则报 404 /upload/image。单段一口气上限 ~5–7 秒，更长的台词拆句/分镜。
7

合成整集

点 「③ 出片并合成」工具 assemble_episode

已出片的镜头自动跳过不重出，直接合成：按镜号拼接 + 普通镜头叠旁白 TTS + 烧字幕 + 对口型保留自带人声 → 一条完整 mp4，落 .agent/video_out/。

合成是纯 ffmpeg(CPU)，不吃 GPU，跟 ComfyUI 开不开无关 → 可以和 GPU 渲染并行做。

▲ 全部出完片 → 点「③ 出片并合成」一键成片

2 · 关键概念

概念说明
旁白 vs 字幕旁白=声音(画外音/台词)；字幕=屏幕上的字。两者独立，可不一样(片头旁白讲故事、字幕打片名)。
尾帧接续一个分镜拆 N 段连续生成再拼，每段从上段末帧继续 → 做长镜头。可事后「再续一段」。
对口型(S2V)图+语音 → Wav2Vec 对齐口型 + 文本引导运动，一口气整段生成，长度由音频定。
中文自动翻译只翻 FLUX 的 image_prompt；运镜词给 Wan(支持中文)、旁白给 TTS，都不翻。

3 · GPU 使用须知(单卡 24G)

出图(FLUX)/普通出片(Wan)/对口型(S2V) 都打同一张卡。后端有单飞队列：同时点会自动排队串行，不会两个挤一张卡 OOM。
能并行的是 GPU 渲染时你做不吃卡的事：选图、改词、调段数、合成(CPU)。同一时刻只跑一个 GPU 任务，同时点不会更快、但安全。
错峰用卡：① 出图阶段停 ComfyUI(FLUX 独占)；② 对口型阶段开 ComfyUI(它缓存住 ~14G)；普通 Wan 与对口型尽量分开，避免同时占卡 OOM。
别绕过面板在 AutoDL 网页另起任务/另开 SSH 出图 → 队列管不到，会抢卡。

4 · 时长规划(2 分钟版参考)

镜头类型怎么定时长
对口型不填段数，时长由台词配音自动决定(一般 4–7s/句)。
普通用段数，段数 ≈ 想要的秒数 ÷ 2(每段≈2s)。重点镜头(打斗/钩子)多给段。
先按计划出，不够再「再续一段」补，比一开始堆太多省返工。16 镜台词自然配音约 1:20，凑 ≥2 分钟靠普通镜头适当加长。

5 · 常见坑 & 排查(实战踩过的)

现象根因解决
出图牛头不对马嘴 / 全是动漫人像FLUX 读不懂中文已自动翻英文；确认 IMAGE_PROMPT_AUTOTRANSLATE 开着
出图全是同一个古装角色套了别人的 LoRA/触发词.agent/config.json 设 trigger_word:""、flux_lora:"none"
视频只有 2 秒单段 Wan ≈2s段数填 3-4，或「再续一段」
对口型台词说不完/被截旧版写死 81 帧≈5s已修：帧数跟音频走；超 ~7s 的句子拆短
对口型失败 404 /upload/imageComfyUI 没启动 / 地址失效tmux 起 ComfyUI；地址变了改 COMFYUI_BASE_URL 后重启后端
改了配置不生效后端启动时才加载 .env改 .env 要重启后端；改分镜词/config.json 不用(现读)
一边出图一边出片会卡崩吗单飞队列已串行不会 OOM，会排队；只是不会更快
一句话速记
写中文 → 出图(自动翻英文)选图 → 普通镜头用段数、说话镜头勾对口型 → 一键合成。
出图阶段停 ComfyUI，对口型阶段开 ComfyUI；一张卡，任务会自动排队。
