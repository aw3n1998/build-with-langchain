# AgentLab 视频生成教学（小说 → 短剧）

> 从一句剧情到一条完整短剧 mp4，AgentLab 的视频生成全流程。
> 每一步：**做什么 / 在哪点 / 填什么 / 背后原理 / 怎么避坑**。文档基于实战跑通的当前版本。

---

## 0. 开工前：环境三件套

| 组件 | 启动方式 | 说明 |
|---|---|---|
| **远程 GPU（4090）** | AutoDL 实例开机 | 出图(FLUX)、普通出片(Wan/LTX) 走 SSH 打这张卡 |
| **ComfyUI**（对口型用） | SSH 进服务器：`tmux new-session -d -s comfy "cd /root/autodl-tmp/comfyui && /root/autodl-tmp/miniconda3/bin/python3.12 main.py --listen 0.0.0.0 --port 6006 > /root/autodl-tmp/comfyui_run.log 2>&1"` | **只有对口型镜头要它**；普通出图/出片不需要 |
| **AgentLab 后端** | `python -m uvicorn agent_lab.main_api:app --host 127.0.0.1 --port 8000` | 改过 `.env` **必须重启**才生效 |
| **前端** | `npm --prefix frontend run dev` | 默认 5173 |

> **实例每次开关，SSH 端口和 ComfyUI 公网地址都会变** → 改 `.env` 的 `GPU_SSH_PORT` 和 `COMFYUI_BASE_URL`，然后**重启后端**。

## 1. 全流程一图

```
建项目 → (配角色) → 加分镜 → 出图 → 选图 → 出片 → 合成整集
                                   │           │
                          中文自动翻英文    ┌── 普通镜头：Wan i2v（段数=尾帧接续）
                          (FLUX 读不懂中文)  └── 对口型镜头：Wan2.2-S2V（台词→TTS→口型）
```

整条都在**制作面板**里点按钮完成（确定性流程，不绕 Agent 对话）。

## 2. 每一步详解

### 第 1 步 · 建项目
- 对 Agent 说「建一个短剧项目，叫《xxx》」，或面板入口新建 → 进**制作面板**。
- 工具：`create_video_project`。

### 第 2 步 · 配角色（可选，但强烈建议）
- 短剧最怕主角每镜长得不一样。有人物 LoRA 就配：「配置角色：触发词 xxx，LoRA 用 <GPU上LoRA路径>」。
- 工具：`configure_character(trigger_word, flux_lora)` → 写入工作目录 `.agent/config.json`，出图自动注入。
- **没训练 LoRA**：留空即可（`trigger_word=""`、`flux_lora="none"`），靠 image_prompt 里的外貌描述压一致性，会有漂移但能用。
- ⚠️ **别套别人的 LoRA**：若 config 里残留了别的角色触发词/LoRA，出图会被强行带成那个角色（跟你提示词无关）。

### 第 3 步 · 加分镜
每镜 6 个字段：

| 字段 | 含义 | 喂给谁 |
|---|---|---|
| `标题` | 镜头名 | — |
| `image_prompt` | **出图**提示词（画面长什么样） | FLUX |
| `motion_prompt` | **运镜/动态**（镜头怎么动） | Wan i2v |
| `narration` | **旁白/台词** | 普通镜头=画外音配音；对口型镜头=人物要说的话 |
| `subtitle` | **屏幕字幕** | 烧屏（留空=沿用旁白） |
| `对口型` | 这镜人物开不开口 | 开=走 S2V，关=普通运镜 |

工具：`add_scene(scene_number, title, narration, image_prompt, motion_prompt, subtitle)`。

### 第 4 步 · 出图（FLUX）
- 每镜点「**出图**」→ 弹**参数卡**（张数/步数/guidance/尺寸/seed/显存策略）→ 改好点出图。
- **防抽卡**：一次出 4 张挑最好的；竖屏短剧尺寸 768×1024。
- 🔑 **中文自动翻英文（隐形）**：FLUX-dev 读不懂中文，你写中文 image_prompt，系统**出图前自动翻成英文**再下发，你无需手动翻。开关 `IMAGE_PROMPT_AUTOTRANSLATE`（默认开）。
- 出图阶段 GPU 吃得满（FLUX 压线 24G）→ **建议先停掉 ComfyUI** 独占整卡。

### 第 5 步 · 选图
- 每镜出 N 张候选 → 点最满意那张的 **✓**（`select_candidate`）。
- 不满意：「**重出图**」换种子再抽；或「**上传图片**」用你自己的图。

### 第 6 步 · 出片 —— 按镜头类型分两种

#### (a) 普通镜头（对口型=否）→ Wan2.2 图生视频
- 选好图 → 行上**段数**填 N → 点「**出视频**」。
- **段数 = 尾帧接续**：单段 Wan-TI2V-5B ≈ **2 秒**（模型特性，单段帧数 ≤25 才稳），N 段用上段末帧续接拼成 ≈2N 秒。
- 不够长 → 成片下方「**➕ 再续一段**」，看效果反复加。
- 不会写运镜 → 「**✨AI 推荐**」，据上段末帧给一句可改的续段运镜（中/英都行，Wan 支持中文）。

#### (b) 对口型镜头（对口型=是）→ Wan2.2-S2V
- 确认这镜 `narration` 写的是**台词**（人物要说的话）。
- 勾「🗣 对口型」→ 按钮变「**对口型出片**」→ 点它。
- 后台自动：台词 → TTS 配音 → 喂 S2V → 出**嘴型跟声音同步**的片（成片自带人声）。
- 🔑 **时长跟着台词走，不截断**：系统按 TTS 音频实际秒数算帧数（`帧数=⌈秒数×fps⌉`，取 Wan 合法的 4n+1），台词多长视频多长，口型对到最后一个字。
- ⚠️ **单段一口气上限 ~5–7 秒**（24G/14B）。超 ~7s 的台词系统会**告警建议拆句/分镜**。
- ⚠️ **必须先启动 ComfyUI**（见第 0 步），否则报 `404 /upload/image`。第一条慢（~2–3 分钟含载模型），之后快。

### 第 7 步 · 合成整集
- 全部出完片 → 面板「**③ 出片并合成**」（已 COMPLETED 的镜头**自动跳过、不重出**，直接合成）；或对 Agent 说「合成整集」。
- 工具：`assemble_episode`。自动：**按镜号拼接 + 普通镜头叠旁白 TTS + 烧字幕 + 对口型镜头保留自带人声**（不重配）→ 一条完整短剧 mp4，落 `.agent/video_out/`。
- 合成是**纯 ffmpeg（CPU）**，不吃 GPU，跟 ComfyUI 开不开无关。

## 3. 关键概念

- **旁白 vs 字幕**：旁白(narration)=声音（普通镜头当画外音配音、对口型镜头当台词）；字幕(subtitle)=屏幕上的字。两者独立，可不一样（比如片头旁白讲故事、字幕打片名）。
- **尾帧接续**：把一个分镜拆成 N 段连续生成再拼，每段从上段末帧继续 → 做长镜头。段数可随时调、也能事后「再续一段」。
- **对口型(S2V)原理**：图 + 语音 → Wav2Vec 音频编码器对齐口型 + 文本引导全局运动，**一口气整段生成**（不是尾帧接续拼的），长度由音频定。
- **中文自动翻译**：只翻 **FLUX 的 image_prompt**；运镜词给 Wan（支持中文）、旁白给 TTS，都不翻。

## 4. GPU 使用须知（单卡 24G）

- **出图(FLUX) / 普通出片(Wan) / 对口型(ComfyUI S2V) 都打同一张卡。**
- 后端有**单飞 GPU 队列**：同时点多个任务**会自动排队串行**，不会两个挤一张卡 OOM。所以**可以一边选图/改词/合成，一边让 GPU 排队渲染**——安全，但同一时刻只跑一个，不会更快。
- **错峰用卡（推荐）**：
  1. **出图阶段**：停掉 ComfyUI，FLUX 独占整卡出完所有图；
  2. **对口型阶段**：启动 ComfyUI 出所有 S2V 镜头（ComfyUI 会缓存住 ~14G 模型）；
  3. 普通出片 Wan 与对口型尽量分开，避免 ComfyUI 占着显存时再跑 Wan → OOM。
- **别绕过面板**手动在 AutoDL 网页另起 ComfyUI 任务/另开 SSH 出图 → 那是面板队列管不到的，会抢卡 OOM。

## 5. 时长规划（2 分钟版参考）

- **对口型镜头不填段数**：时长由台词配音自动决定（一般 4–7s/句）。
- **普通镜头用段数**：段数 ≈ 想要的秒数 ÷ 2（每段 ≈2s）。
- 16 镜的台词自然配音约 1 分 20 秒，要凑 ≥2 分钟，普通镜头适当加长（重点镜头如打斗/钩子多给段）。先按表出，**不够再「再续一段」补**，比一开始堆太多省返工。

## 6. 常见坑 & 排查（都是实战踩过的）

| 现象 | 根因 | 解决 |
|---|---|---|
| 出图牛头不对马嘴 / 全是动漫人像 | FLUX-dev 读不懂中文 | 已**自动翻英文**；若还不对，确认 `IMAGE_PROMPT_AUTOTRANSLATE` 开着 |
| 出图全是同一个古装角色 | 工作目录套了别人的 LoRA/触发词 | `.agent/config.json` 设 `trigger_word:""`、`flux_lora:"none"` |
| 视频只有 2 秒 | 单段 Wan ≈2s | 段数填 3-4，或「再续一段」 |
| 对口型台词说不完/被截 | 旧版写死 81 帧≈5s | 已修：帧数跟音频走；超 ~7s 的句子拆短 |
| 对口型出片失败 `404 /upload/image` | ComfyUI 没启动 / 公网地址失效 | SSH 用 tmux 起 ComfyUI；地址变了改 `.env` 的 `COMFYUI_BASE_URL` 后**重启后端** |
| 改了配置不生效 | 后端启动时才加载 `.env`/settings | 改 **`.env`** 要**重启后端**；改分镜提示词/`.agent/config.json` 不用（现读） |
| 一边出图一边出片会不会卡崩 | 单飞队列已串行 | 不会 OOM，会排队；但同一时刻只跑一个，不会更快 |

## 7. 一句话速记

> **写中文 → 出图(自动翻英文)选图 → 普通镜头用段数、说话镜头勾对口型 → 一键合成。**
> **出图阶段停 ComfyUI，对口型阶段开 ComfyUI；一张卡，任务会自动排队。**
